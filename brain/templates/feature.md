# Feature Template

## Purpose
This file is the starter template for `brain/features/<feature>.md`.

## How To Use
- Copy this structure for each feature doc.
- Keep implementation notes factual and current.

## Template
```md
# <Feature Name>

## Purpose
Short description of the feature.

## Goal
What user or business outcome this feature delivers.

## Scope
- Included
- Excluded

## User Flow
1. Step
2. Step

## Data Model
- Entity
- Relationship

## APIs
- Route or procedure

## UI
- App surfaces

## Edge Cases
- Case

## Permissions
- Role behavior

## Future Improvements
- Improvement
```
